AI TIMETABLE BACKEND - POSTMAN MANUAL TESTING GUIDE
Date: 2026-03-11

BASE URL
- Local: http://127.0.0.1:8000

POSTMAN ENVIRONMENT VARIABLES (recommended)
- base_url = http://127.0.0.1:8000
- access_token = (empty initially)
- refresh_token = (empty initially)
- entry_id = 1
- conversation_id = 1
- notification_id = 1
- uid = (from password reset email log)
- reset_token = (from password reset email log)

COMMON HEADERS FOR POSTMAN

1) Public JSON endpoints (no login needed)
- Accept: application/json
- Content-Type: application/json

2) Protected JSON endpoints (login required)
- Accept: application/json
- Content-Type: application/json
- Authorization: Bearer {{access_token}}

3) Protected file upload endpoint (OCR image)
- Accept: application/json
- Authorization: Bearer {{access_token}}
- In Postman Body use form-data, key exam_image (type File)
- Do not manually force Content-Type for form-data

ALL API ROUTES (METHOD + PATH)

AUTH
- POST /api/auth/register/
- POST /api/auth/token/
- POST /api/auth/token/refresh/
- POST /api/auth/logout/
- POST /api/auth/password-reset/
- POST /api/auth/password-reset/confirm/<uidb64>/<token>/
- GET  /api/auth/me/

TIMETABLE
- POST /api/timetable/chatbot/
- GET  /api/timetable/entries/
- GET  /api/timetable/entries/<pk>/
- PATCH /api/timetable/entries/<pk>/
- PUT /api/timetable/entries/<pk>/
- POST /api/timetable/entries/<pk>/completion-response/
- GET  /api/timetable/notifications/
- PATCH /api/timetable/notifications/<pk>/read/

CHATBOT
- POST /api/chatbot/converse/
- GET  /api/chatbot/conversations/
- GET  /api/chatbot/conversations/<conversation_id>/messages/

ADMIN
- /admin/ (Django admin, usually browser/session auth, not JWT API style)

============================================================
MANUAL TESTING FLOW (recommended order)
============================================================

STEP 1: REGISTER USER
Request
- Method: POST
- URL: {{base_url}}/api/auth/register/
- Headers: Public JSON headers
Body (raw JSON)
{
  "email": "manualtester@example.com",
  "password": "Test12345",
  "first_name": "Manual",
  "last_name": "Tester"
}
Expected
- Status: 201 Created (or 200 in some setups)
- Response example:
{
  "id": 1,
  "email": "manualtester@example.com",
  "first_name": "Manual",
  "last_name": "Tester"
}

STEP 2: LOGIN (GET TOKENS)
Request
- Method: POST
- URL: {{base_url}}/api/auth/token/
- Headers: Public JSON headers
Body
{
  "email": "manualtester@example.com",
  "password": "Test12345"
}
Expected
- Status: 200 OK
- Response example:
{
  "refresh": "<jwt_refresh_token>",
  "access": "<jwt_access_token>"
}
Postman Tests tab (optional):
const data = pm.response.json();
pm.environment.set("access_token", data.access);
pm.environment.set("refresh_token", data.refresh);

STEP 3: GET MY PROFILE
Request
- Method: GET
- URL: {{base_url}}/api/auth/me/
- Headers: Protected JSON headers
Expected
- Status: 200 OK
- Response example:
{
  "id": 1,
  "email": "manualtester@example.com",
  "first_name": "Manual",
  "last_name": "Tester"
}

STEP 4: TOKEN REFRESH
Request
- Method: POST
- URL: {{base_url}}/api/auth/token/refresh/
- Headers: Public JSON headers
Body
{
  "refresh": "{{refresh_token}}"
}
Expected
- Status: 200 OK
- Response has a new access token, and may also return refresh if rotation is enabled.
Example:
{
  "access": "<new_access_token>",
  "refresh": "<maybe_new_refresh_token>"
}

STEP 5: PASSWORD RESET REQUEST
Request
- Method: POST
- URL: {{base_url}}/api/auth/password-reset/
- Headers: Public JSON headers
Body
{
  "email": "manualtester@example.com"
}
Expected
- Status: 204 No Content
- No response body
Notes
- This backend uses console email backend in development.
- Check the terminal running Django for reset link containing uid and token.

STEP 6: PASSWORD RESET CONFIRM
Request
- Method: POST
- URL: {{base_url}}/api/auth/password-reset/confirm/{{uid}}/{{reset_token}}/
- Headers: Public JSON headers
Body
{
  "uid": "{{uid}}",
  "token": "{{reset_token}}",
  "new_password": "Newpass12345"
}
Expected
- Status: 200 OK
- Usually empty body
Important
- Current view validates uid/token from JSON body fields.

STEP 7: SAVE TOPICS + FREE SLOTS AND GENERATE TIMETABLE
Request
- Method: POST
- URL: {{base_url}}/api/timetable/chatbot/
- Headers: Protected JSON headers
Body
{
  "topics": [
    {
      "name": "Math",
      "estimated_minutes": 120,
      "priority": 2
    },
    {
      "name": "Operating Systems",
      "estimated_minutes": 90,
      "priority": 3

    {
      "start": "2026-03-15T10:00:00Z",
      "end": "2026-03-15T12:00:00Z"
    },
    {
      "start": "2026-03-16T09:00:00Z",
      "end": "2026-03-16T10:30:00Z"
    }
  ]
}
Expected
- Status: 200 OK
- Response example (array):
[
  {
    "id": 1,
    "topic": {
      "id": 1,
      "name": "Math",
      "estimated_minutes": 120,
      "priority": 2,

]
Validation notes
- Duplicate topic name for same user can return 400.
- Overlapping free slots can return 400.

STEP 11: SEND COMPLETION RESPONSE FOR AN ENTRY
Request
- Method: POST
- URL: {{base_url}}/api/timetable/entries/{{entry_id}}/completion-response/
- Headers: Protected JSON headers
Body (completed false example)
{
  "completed": false,

  "quiz_answer": "I revised only half of the topic"
}
Expected
- Status: 200 OK
- Response example (rescheduled path):
{
  "status": "rescheduled",
  "entry_done": false,
  "completion_check": {
    "entry_id": 1,
    "asked_at": "2026-03-15T12:10:00Z",
    "response_received_at": "2026-03-15T12:20:00Z",
    "completed": false,
    "quiz_question": "Quick check question"
  },
  "strategy": {
    "action": "split_topic_into_smaller_sessions",
    "max_chunk_minutes": 30,
    "priority_boost": 1,
    "extra_minutes_ratio": 0.0
  },
  "generation": {
    "algorithm": "score_weighted_exam_aware"
  },
  "entries": [
    {
      "id": 2,
      "topic": "Math",
      "topic_id": 1,
      "start": "2026-03-16T09:00:00Z",
      "end": "2026-03-16T09:30:00Z",
      "done": false
    }
  ]
}
Body (completed true example)
{
  "completed": true,
  "response_text": "Completed successfully"
}
Expected
- Status: 200 OK
- Response has status "completed" and entry_done true

============================================================
COMMON ERROR RESPONSES FOR QUICK DEBUG
============================================================

1) Missing/invalid JWT
- Status: 401
{
  "detail": "Authentication credentials were not provided."
}
or
{
  "detail": "Given token not valid for any token type",
  "code": "token_not_valid"
}

2) Duplicate topic name in /api/timetable/chatbot/
- Status: 400
{
  "name": ["Topic with this name already exists."]
}

3) Overlapping free slot in /api/timetable/chatbot/
- Status: 400
{
  "non_field_errors": ["Free slot overlaps with an existing slot."]
}

4) Missing payload for explicitly selected chatbot tool
- Status: 400
{
  "error": "Tool '<tool_name>' missing required payload.",
  "tool": "<tool_name>"
}

5) Empty chatbot request
- Status: 400
{
  "error": "No actionable input provided.",
  "hint": "Send message/onboarding/generate_timetable/exam_image/tool."
}
